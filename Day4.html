Understanding the File Path

As we know that we can add image in our website using img tag in HTML and by determining it's source by using src.

But what we will do when we have to add an image that is present inside our system only then how we will determine the path of that image.

Here comes the role of File Path.

You can determine the path of the image that is stored inside your system itself by two ways:

  1. Relative Path

    Relative path is determined from the current position.

    For Example:

    Suppose you have 3 folders in your system. Inside each folder there are three files.

    1. Folder1 is Alpha
      • File1.1 is dog.png
      • File1.2 is cow.png
      • File1.3 is ex.html
    2. Folder2 is Beta which is inside folder1
      • File2.1 is lion.png
      • File2.2 is cat.png
      • File2.3 is x.html
    3. Folder3 is Gamma which is inside folder2
      • File3.1 is pet.png
      • File3.2 is rabbit.png
      • File3.3 is xe.html

      a. Now let's suppose you are in file2.3 which is inside folder2 and you want to add the image stored in file 2.2

      b. You can acheive the above task by simply writing the name of that file that is cat.png or ./cat.png in src.

      c. Now if you want to add the image which is not in the folder2 but in folder3 which is inside folder2.

      d. So now you have to write like this in src.

      Folder3/pet.png

      You can also write like this ./Folder3/pet.png

      f. Now if you want to add the image which is not in the folder2 but in folder1 inside which folder2 is there.

      g. So now you have to write like this in src.

      ../cow.png

      4. h. In relative path you can go up and as well as down from your current position.

  2. Absolute Path
    • In absolute path you give exact path of the file from core or root level.
    • For Window's Absolute path looks like C:\Users\John\Documents\report.docs.
    • For Linux Absolute path looks like Users/John/Documents/report.docs

Why we don't use absolute path most of the time?

Because it gives another system to open particular file from the root level of your system. Which is undoubtedly very dangerous for your system as it contains sensitive data as well.

That is why browser automatically block the absolute path to protect your system.

Same apply for relative path as well if you give the relative path in such a way that it can enter's inside your system then in that case as well browser blocked it's path.

What is div tag?

  • Div is like the container inside which their are multiple tags.
  • Div is used when we want to make changes in multiple elements which are continuous in code.
  • Instead of making changes in them one by one we put those elements in div container.
  • And then make changes in div container ultimately leads to the changes in those elements as well

Concept of class And id

class

  • Class is used when you want to make changes in elements or tags which are not continuous in code.
  • You can make changes in them by giving them the same class.
  • Class can be repeated.
  • Changes in class leads to changes in those elements.
  • Class is nothing just the grouping of elements or tags.

id

  • Id is used when you want to make changes in any particular element.
  • Id is used to select element uniquely.
  • Id can not be repeated.
  • Id is unique.
